**图像分割** 是许多计算机视觉应用中的关键处理步骤，通常用于将图像划分为不同的区域，这些区域常常对应于真实世界的对象。因此图像分割是图像识别和内容分析的重要步骤。**阈值分割** 是一种简单、有效的图像分割方法，其中像素根据其强度值进行分区。人们已经走出的路可以分为 3 条：

1. **全局阈值路线**：人工设定 → 迭代求均值 → 自动最优（类间方差最大）

全局阈值的目标都是找到一个最优的阈值 T 来分割图像。最早的阈值分割依靠经验设定全局阈值，后来发展出迭代法（根据前景/背景均值自动更新阈值），再进一步由 大津法/三角形计算法 实现自动化选择最优阈值。

2. **自适应阈值（局部阈值）路线**：先驱（Niblack & Sauvola） → OpenCV（邻域平均、邻域加权平均）

局部阈值最初是为了解决光照不均这一核心问题而诞生的。Niblack/Sauvola 是 “理论先驱和算法发明者”，而 OpenCV 的 adaptiveThreshold 是 “成熟、高效、易用的工业产品”。在很多情况下，Sauvola 算法在效果上（尤其是文档二值化）仍然优于 OpenCV 的自适应阈值，但后者在速度和易用性上优势明显。

3. **信息论路线**：最大熵阈值法

熵方法源于信息论。它将图像分割看作是将图像的灰度直方图分成两个概率分布（前景 PDF 和背景 PDF）。它寻找的阈值，是能使这两个分布所携带的信息量（熵）之和最大的那个点。**直观理解**：一个好的分割，应该让前景和背景都尽可能“丰富”，保留尽可能多的信息。它不希望分割后的任何一个部分变得“平淡”或“无序”。

## 1. 全局阈值分割

### 1.1 简单全局阈值

前面说过，最早是依靠经验值设定全局阈值。也就是说，我们设定的是一个固定数值。OpenCV 提供了 ==cv2.threshold()== 函数接口，用于实现简单阈值处理：

```py
retval, dst = cv2.threshold(src, thresh, maxval, type=xxx)
```

- **返回值 retval**：实际使用的阈值。对于自动算法会返回计算出的最佳阈值；否则就是传入的 thresh。
- **返回值 dst**：阈值化后的图像（与输入图像同尺寸）。
- **thresh**：表示阈值，即分割的临界值，低于/高于 此值的像素如何处理由 type 决定。
- **maxval**：当像素满足阈值条件时被赋予的值（一般是 255），仅在前两个阈值类型有效。
- **type**：阈值类型，决定阈值化的规则（见下表）

| 类型常量                | 行为说明                                      | 图像变化       |
| ----------------------- | --------------------------------------------- | -------------- |
| `cv2.THRESH_BINARY`     | 如果像素值 > 阈值，则设为 maxval，否则设为 0  | 二值化（常见） |
| `cv2.THRESH_BINARY_INV` | 与上面相反：小于阈值的设为 maxval，大于设为 0 | 反二值化       |
| `cv2.THRESH_TRUNC`      | 大于阈值的像素设为阈值，小于的保持不变        | 截断上限       |
| `cv2.THRESH_TOZERO`     | 小于阈值的像素设为 0，大于的保持不变          | 去掉暗部       |
| `cv2.THRESH_TOZERO_INV` | 大于阈值的设为 0，小于的保持不变              | 去掉亮部       |

为了方便理解，我们做一个小实验，构造 6 个不同颜色的灰度像素块，然后用不同的预定义阈值 [0, 50, 100, 150, 200]，对其做一个分割，阈值化规则设定为 THRESH_BINARY：

![1.1 简单全局阈值的理解](/cv/ImageProcessing/06_xxx/01_sample1.png)

是不是很直观，大于阈值的，统统都变成了我们设置的 255 白色，小于阈值的全部都变成了 0 黑色。但是，阈值法是非常理想化的，它认为图像的灰度分布是双峰的，一峰对应背景，一峰对应前景，中间有个“谷”刚好是分界线。

这类图像也有，通常是 白底黑字的文档扫描、黑底白线的显微镜图像、工业检测中亮度单一的金属表面 等等。但是一般的图像，并不是简单的全局阈值就可以分割的，比如我们对芙莉莲的动漫图像进行阈值分割：

### 1.2 迭代法与大津法

在简单全局阈值中，我们人为给定一个阈值 𝑇，将图像分为前景与背景。但对于复杂图像，这种“拍脑袋式”的阈值往往不稳定。为此，人们提出了 ==迭代法（Iterative Threshold Selection）==，这是一种基于统计均值的自适应方式。

**迭代法假设，前景与背景的灰度分布在一定程度上可区分，最优阈值应该接近两者均值的中点**。因此它不断地用当前阈值划分图像，分别计算两部分的平均灰度，更新阈值为二者均值，重复直到收敛。

这其实是一种 “经验迭代优化”，类似于无监督分类（两类聚类）的过程。**取全图的均值，大于均值的定义为前景，小于均值的定义为背景，然后不断迭代（再取全图，也就是 前景 + 背景 的均值）**。

![1.3 全局阈值自动化 - 迭代法](/cv/ImageProcessing/06_xxx/02_Frieren2.png)

迭代法的本质上就是在逼近一个 “类内差最小、类间差最大” 的分割点。==大津法（Otsu’s Method）== 是在迭代法思想上发展出的形式化方法，**一次性在所有可能阈值上搜索，使类间方差最大**。它继承了 “前景背景统计对比” 的核心逻辑，只是换成了方差度量而不是均值更新。

Otsu 在 1979 年的论文《A threshold selection method from gray-level histograms》中提出，将阈值分割问题形式化为一个最值问题。最大化类间方差（Between-Class Variance）：

$$
\sigma^{2}_{B}=w_{1}w_{2}(\mu_{1}-\mu_{2})^{2}=w_{1}(\mu_{1}-\mu_{T})^{2}+w_{2}(\mu_{2}-\mu_{T})^{2}
$$

- $w_{1}$，$w_{2}$ 分别是前景与背景的像素比例。
- $\mu_{1}$，$\mu_{2}$ 是对应灰度的均值。
- $\mu_{T}$ 表示全图平均灰度。

使用起来也很简单，**cv2.threshold()** 函数中的 type 参数加上一个 ==cv2.THRESH_OTSU== 即可。这里需要说明的是，大津法只能与 **THRESH_BINARY** 与 **THRESH_BINARY_INV** 组合使用，因为它本质是生成二值图，其它的方法是截断，不属于二值类型。

```py
cv2.threshold(src, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
cv2.threshold(src, 0, 255, cv2.THRESH_BINARY_INV + cv2.THRESH_OTSU)
```

import cv2
import numpy as np
import matplotlib.pyplot as plt

def show_img_by_plt(img, title, pos):
    ax = plt.subplot(2, 3, pos)
    plt.imshow(img, cmap="gray")
    plt.title(title)
    plt.axis('off')

# ------------ 基于迭代法计算最优全局阈值 --------------
def iterative_threshold(image, eps=0.5):
    """
    image: 灰度图像 (numpy 数组)
    eps: 收敛条件，当新旧阈值差小于该值时停止迭代
    """
    thresholds = []
    T = np.mean(image)
    prev_T = 0

    while abs(T - prev_T) > eps:
        thresholds.append(T)
        prev_T = T
        fg = image[image > T]
        bg = image[image <= T]
        if len(fg) == 0 or len(bg) == 0:
            break
        T = (np.mean(fg) + np.mean(bg)) / 2

    thresholds.append(T)  # 记录最终阈值
    return thresholds
```

### 1.3 三角形计算法

==三角形算法（Triangle Thresholding Method）== 是一种基于直方图几何特性的自动阈值选取方法，由 Zack 等人在 1977 年提出。它特别适用于 **单峰 - 长尾型灰度分布** 的图像（例如显微镜、医学影像、磨损检测等）。它的核心思想是 **最大化几何距离**：

1. **绘制灰度直方图**：横轴为灰度值 (0–255)，纵轴为像素数；
2. **确定两个关键点**：峰值点（直方图最大值所在的灰度）；灰度分布最远一端的端点（根据尾部方向确定）。
3. **计算最佳阈值**：两点连线成一条直线，计算每个点到这条线的垂直距离，距离最大的点对应的灰度值就是 “最佳阈值”。

它要找到一个能 “最大拉开直方图主峰与背景尾部” 的分界点。适合低对比、亮度偏移图像，比如雾天的模糊照片、医学 CT 影相、工业检测、浅色素描或者文稿。类似这样，主峰 + 平滑尾部延伸：

![1.5 单峰 - 长尾型灰度分布](/cv/ImageProcessing/06_xxx/03_sanjiao.png)

是不是感觉限制非常多？确实如此，我们拿 **芙丽莲的原图** 与 **芙丽莲的素描** 做一个峰值的对比。方法很简单，用 ==cv2.calcHist()== 做一个全局直方图统计，再用 **scipy.signal** 库中的 ==find_peaks== 方法寻找峰值。

![1.6 寻找并可视化图像的峰值](/cv/ImageProcessing/06_xxx/03_sanjiao2.png)

可以看到，素描已经很类似 **单峰 - 长尾型分布** 了，它的小区间不平滑是因为，笔触与阴影构成了图像的局部结构特征，可以通过高斯模糊，去除一部分的峰值，但是并不好完全消除。

三角形计算法使用起来也很简单，与大津法一样只能与 **THRESH_BINARY** 与 **THRESH_BINARY_INV** 组合使用，**cv2.threshold()** 函数中的 type 参数加上一个 ==cv2.THRESH_TRIANGLE== 即可。我们再看看三角形计算法下的两者对比，这次就更明显了：

import cv2
import numpy as np
import matplotlib.pyplot as plt
from scipy.signal import find_peaks

def show_hist(img, title, pos=None):
    ax1 = plt.subplot(2, 2, pos)
    plt.imshow(img, cmap="gray")
    plt.title(title)
    plt.axis('off')

    # 右图：绘制图像的灰度直方图，并尝试检测是否为单峰长尾分布
    hist = cv2.calcHist([img], [0], None, [256], [0,256]).ravel()
    peaks, _ = find_peaks(hist, distance=20)

    ax2 = plt.subplot(2, 2, pos+1)
    plt.plot(hist, color='black')
    plt.title("Gray Histogram")
    plt.xlabel("Gray Level")
    plt.ylabel("Pixel Count")

    # 标记峰值
    for p in peaks:
        plt.axvline(p, color='r', linestyle='--', alpha=0.6)

    # 判断单峰长尾
    if len(peaks) == 1:
        peak_pos = peaks[0]
        left_tail = np.sum(hist[:peak_pos]) / np.sum(hist)
        right_tail = np.sum(hist[peak_pos:]) / np.sum(hist)
        skew = right_tail - left_tail
        if skew > 0.1:
            print("👉 图像直方图为：单峰 + 右长尾（亮区偏多）")
        elif skew < -0.1:
            print("👉 图像直方图为：单峰 + 左长尾（暗区偏多）")
        else:
            print("👉 图像直方图为：单峰对称分布")
    else:
        print(f"⚠️ 检测到 {len(peaks)} 个峰值，可能不是单峰分布。")


img1 = cv2.imread("04_fulilian.jpg", 0)
img2 = cv2.imread("04_fulilian1.jpg", 0)
plt.figure(figsize=(12,5))
show_hist(img1, "Original Gray Fulilian", pos=1)
show_hist(img2, "Sketch Fulilian", pos=3)
plt.show()

## 2. 局部阈值分割

### 2.1 自适应阈值（先驱）

在计算机视觉发展早期，研究者就发现：**全局阈值分割在光照不均或背景变化的情况下极易失败**，于是上世纪 1980–1990 年间，出现了一系列以局部统计信息为基础的阈值计算方法。

其中最具代表性的两种就是 **Niblack (1986)** 和 **Sauvola (1997)**。这两种算法构成了后来自适应阈值的理论基础，也被称为 “局部阈值分割的先驱方法”。

==Niblack 算法== 的思想非常直观：**每个像素点的阈值应由它邻域的平均值和标准差共同决定**。在一个以像素 (x, y) 为中心，大小为 w x w 的邻域中，数学公式定义为：

$$
T(x,y)=m(x,y)+k\cdot{s(x,y)}
$$

- **w**：局部窗口大小，通常为 15 ~ 51，奇数。
- **k**：调节常数，控制偏向性，一般取 -0.2 ~ -0.1。
- $m(x,y)$ 与 $s(x,y)$：表示 邻域均值 和 邻域标准差。

当局部标准差 $s(x,y)$ 较大时（即边缘或噪声处），阈值 T 下降，利于把较暗的细节与较亮的内容分开。当区域平坦（灰度差小）时，阈值上升，抑制错误分割。因此该方法具有较好的局部自适应能力，**但对噪声敏感**。

![2.1 自适应阈值 - Niblack](/cv/ImageProcessing/06_xxx/04_Niblack.png)

==Sauvola 算法== 在 Niblack 的基础上改进了其“阈值易偏高”的缺陷，引入了灰度动态范围 𝑅 的归一化，使算法更加稳定：

$$
T(x,y)=m(x,y)+\left[1+k\left(\frac{s(x,y)}{R}-1\right)\right]
$$

- **𝑅**：标准差的动态范围（一般取 128）。
- **k**：调节常数，典型值为 0.2 ~ 0.5。

![2.2 自适应阈值 - Sauvola](/cv/ImageProcessing/06_xxx/04_Sauvola.png)

**Sauvola 算法** 的效果还是很不错的，这也是为什么它到现在还在被使用。但这个结果有没有一种 **锐化** 的感觉？确实是有的哈，没感觉错。Sauvola 让高对比区域更亮，低对比区域更暗，造成了一种视觉上的 “边缘强化感”。但 Sauvola 在二值决策阈值层面增强对比度，而锐化在像素灰度层面增强对比度，本质还是不同的。

# ================ 使用盒滤波高效计算局部均值与标准差 =================
def _local_mean_std(img_gray, win_size):
    if win_size % 2 == 0 or win_size < 3:
        raise ValueError("win_size 必须为 >=3 的奇数")
    f = img_gray.astype(np.float32)  # 转换为 float32，返回值也为 float32

    # 局部均值 E[X]，局部平方均值 E[X^2]
    mean = cv2.boxFilter(f, ddepth=-1, ksize=(win_size, win_size),
                         normalize=True, borderType=cv2.BORDER_REFLECT)
    mean_sq = cv2.boxFilter(f * f, ddepth=-1, ksize=(win_size, win_size),
                            normalize=True, borderType=cv2.BORDER_REFLECT)

    # 方差 Var[X] = E[X^2] - (E[X])^2，标准差：再开方
    std = cv2.sqrt(cv2.max(mean_sq - mean * mean, 0))
    return mean, std
```

### 2.2 扩展回顾：盒滤波

==盒滤波（Box Filter）== 是一种 **均值滤波器（Mean Filter）** 的实现形式，在《噪声与平滑化》中有提到过。它的核心思想是，在一个固定大小的矩形窗口中，对所有像素取平均。数学形式为：

$$
g(x,y)=\frac{1}{w\cdot{h}}\sum^{a}_{i=-a}\sum^{b}_{j=-b}f(x+i,y+j)
$$

也就是说 每个像素的输出 = 它邻域内所有像素灰度的平均；卷积核内的权值全都相等（看起来像一个“盒子”）。实际上在 OpenCV 内部，**cv2.blur()** 的底层实现正是调用 ==cv2.boxFilter()==。

```py
dst = cv2.boxFilter(src, ddepth, ksize, normalize=True, borderType=cv2.BORDER_DEFAULT)
```

需要注意的是我们上一小节用了 **borderType = cv2.BORDER_REFLECT**，它是指边缘镜面反射（不包含边缘自身），是为了在局部计算时不错算边缘。这个参数在《基础操作》中也讲过。

盒滤波除了我们的 **均值滤波**，用于平滑去噪之外，常见的应用也很广：

1. **局部统计量计算（均值/方差/标准差）**：比如我们刚才的局部阈值分割，再比如局部对比度增强。
2. **积分图运算中的平均值计算**：用于加速大窗口卷积；在特征提取（如 Haar-like 特征）中大量使用。boxFilter 可以用积分图实现 O(1) 均值计算。
3. **导向滤波和双边滤波**：在这些复杂滤波的推导中，局部均值运算通常用 box filter 实现，特别是在局部均值项的快速近似版本中。
4. **图像金字塔（Pyramid）生成**：构建 Gaussian Pyramid 时有一类快速实现（均值代替高斯），用于金字塔模糊、特征层缩放。

### 2.3 自适应阈值（OpenCV）

OpenCV 提供了 ==cv2.adaptiveThreshold()== 函数来实现自适应阈值。

```py
dst = cv2.adaptiveThreshold(src, maxValue, adaptiveMethod, thresholdType, blockSize, C)
```

- **src**：输入图像，必须是 uint8 的单通道图像，一般都是彩色转灰度。
- **maxValue**：满足条件的像素赋值（通常为 255），类似全局阈值中的 maxval。
- **adaptiveMethod**：自适应方式。
- **thresholdType**：阈值类型，常用 THRESH_BINARY 或 THRESH_BINARY_INV。
- **blockSize**：邻域大小（奇数，≥3），通常 11 ~ 55。
- **C**：常数偏移量，调节阈值高低，通常取 2 ~ 10。正值的结果更暗；负值的结果更亮。

关键参数为 adaptiveMethod，OpenCV 提供了两种计算方式。==cv2.ADAPTIVE_THRESH_MEAN_C== 表示**均值法**，它其实是 Niblack 算法的简化版：

$$
T(x,y)=mean(x,y)-C
$$

C 是一个手动设定的偏移值常数。从公式可以看出，它保留了局部均值，但不考虑局部方差。但是从结果可以看出，它的效果要比 Niblack 好。并且在处理前，用高斯滤波进行平滑处理后，效果会更好。原图不明显，但是素描图的结果很明显。

### 3.1 最大熵阈值

==最大熵阈值法（Maximum Entropy Thresholding）== 是一种基于信息论的图像分割方法。它通过最大化前景和背景的熵（信息量），来确定最优的分割阈值，使得分类后的信息不确定性最大，也即信息量最丰富。

它的优点是能更好应对双峰或多峰灰度分布的情况；缺点是计算量略大，对噪声较敏感。数学公式推导：

1. **定义灰度级概率分布为（N 表示像素总数，$n_{i}$ 表示不同灰度级所属的像素数量）**：

$$p_{i}=\frac{n_{i}}{N}$$

2. **假设阈值为 T，则前景灰度区间为 [0, T]，背景灰度区间为 [T+1, L-1]，从概率分布上来说前景与背景可以写为（L 为灰度级数，一般为 256）**:

$$P_{1}=\sum^{T}_{i=0}p_{i},\quad P_{2}=\sum^{L-1}_{i=T+1}p_{i}$$

3. **计算前景与背景的熵**。它的理论来源是 ==香农信息熵（Shannon Entropy）==，我们先从它看起。信息论中，一个离散随机变量 X 的信息熵定义为：

$$H(X)=-\sum_{i}p_{i}\log{p_{i}}$$

其中 $P_{i}$ 表示事件 i 出现的概率，$H(X)$ 是描述系统不确定性（信息量）的指标。举个简单例子，如果所有事件概率都相等（最“混乱”），熵最大；如果某个事件概率为 1（完全确定），熵为 0。

延伸到图像阈值分割，灰度图像也可以看作一种概率分布系统，每个灰度级 i 的出现频率就是概率 $p_{i}$。但是别忘了，我们切分了前景与背景两个系统，所以 **需要分别计算两者的局部熵（$p_{i}$ 在 [0, 1] 区间，所以 log 是负的，前方加的 “负号” 是为了抵消它）**：

$$
H_{1}=-\sum^{T}_{i=0}\frac{p_{i}}{P_{1}}\log{\frac{p_{i}}{P_{1}}},\quad
H_{2}=-\sum^{L-1}_{i=T+1}\frac{p_{i}}{P_{2}}\log{\frac{p_{i}}{P_{2}}}
$$

4. **最大化目标函数，当整体信息熵最大时，得到的 T 即为最优阈值（arg 表示 argument of，$H_{1}+H_{2}$ 表示总熵，取总熵最大值时的阈值 T）**：

$$T^{*}=argmax_{T}(H_{1}+H_{2})$$

公式的推导到这里就结束了，我们来实现一下算法。在这里我们用 Numpy 的 ==np.histogram()== 函数来计算直方图概率。OpenCV 的直方图 **cv2.calcHist()** 也可以用，但是通常是为了可视化/均衡化使用的，Numpy 会更直接、更轻量一点。

:::: code-group
::: code-group-item 最大熵阈值 - 计算函数的实现

```py
import cv2
import numpy as np
import matplotlib.pyplot as plt

# -------- 最大熵阈值法函数 --------
def max_entropy_threshold(img):
    # 统计图像中每个灰度值出现的次数，也就是计算图像的直方图。
    # hist 是长度为 256 的数组，每个元素是对应灰度的像素数；img.ravel() 把图像转为 1 维
    hist, _ = np.histogram(img.ravel(), bins=256, range=(0, 256))
    hist = hist.astype(np.float32)
    hist /= hist.sum()  # 归一化操作，等于算法第一步 p_i = n_i / N

    max_ent, threshold = 0, 0
    for t in range(256):
        P1 = np.sum(hist[:t+1])
        P2 = np.sum(hist[t+1:])
        if P1 == 0 or P2 == 0:
            continue
        # +1e-12 只是为了防止 log(0) 出现无穷大错误；不影响结果，是一个数值安全保护。
        H1 = -np.sum((hist[:t+1]/P1) * np.log(hist[:t+1]/P1 + 1e-12))
        H2 = -np.sum((hist[t+1:]/P2) * np.log(hist[t+1:]/P2 + 1e-12))
        H = H1 + H2
        if H > max_ent:
            max_ent, threshold = H, t
    return threshold
```

### 3.2 彩色图像阈值处理

彩色图像有三个通道：R、G、B。常见的做法是 **对每个通道独立执行灰度阈值分割**。然而大多数经典算法都是针对单通道灰度图像推导的。现实中的彩色图像信息更丰富，单通道分割很多时候分不出有用区域。记录这种处理方式，只是说明 **阈值法的思路可以自然扩展到多通道情况**。比如：

1. **印章 / 签名提取**：红色或蓝色区域在 RGB 某一通道上阈值明显可分；可快速提取出印章。
1. **病理切片分析**：染色区域（H&E 染色的紫色核与粉红细胞质）在各通道差异明显；可阈值区分组织结构。
1. **农作物分割 / 叶片检测**：植物绿叶在 G 通道分布明显集中；简单阈值能快速提取前景。
1. **交通标志识别 / 车道线检测**：颜色分布清晰的红、黄区域用通道阈值法提取比灰度快。
