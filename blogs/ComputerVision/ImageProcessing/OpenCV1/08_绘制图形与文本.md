我们在《形态学与运算》中用圆形和字母演示过基本图形的生成；在《边缘与轮廓检测》中，我们又用了矩形和圆形。这些都是 OpenCV 绘图功能的直接应用场景 - 在图像上可视化算法结果。在计算机视觉任务中，绘制图形与文本的常见用途包括：

- **目标检测**：用矩形框（cv2.rectangle）标出检测目标；
- **人脸识别 / 关键点检测**：用圆点或连线（cv2.circle、cv2.line）表示关键点；
- **语义分割 / 分类结果展示**：在区域或边缘绘制轮廓线；
- **结果可视化与标注**：用 cv2.putText 添加类别、置信度、标识信息。

相比 Pillow，OpenCV 的绘制功能虽然语法更底层、功能稍简，但在 实时视频流、检测结果展示、或与 OpenCV 算法链集成 时极为高效。因为 OpenCV 的底层是 C++ 实现的，性能比 Pillow 更好。

## 1. 绘制文本

### 1.1 颜色设定

OpenCV 默认采用 ==BGR 顺序（蓝、绿、红）==，而不是 RGB，所以 (255, 0, 0) 表示 蓝色，而不是红色。在需要与 Matplotlib / Pillow 等库互通时，可用色彩空间转换函数：

```py
cv2.cvtColor(img, cv2.COLOR_BGR2RGB)  # 色彩空间转换
```

不管是绘制图形，还是绘制文本，颜色都是用的 ==BGR== 三原色的元组，因此当我们想手动定义颜色时，推荐字典写法。这种方法是现代且易维护的，比写一堆常量可读性更高：

```py
import random

# 颜色字典定义，红橙黄绿青蓝紫，黑白随机
COLORS = {
    'red': (0, 0, 255),
    'orange': (0, 165, 255),
    'yellow': (0, 255, 255),
    'green': (0, 255, 0),
    'cyan': (255, 255, 0),
    'blue': (255, 0, 0),
    'purple': (255, 0, 255),
    'black': (0, 0, 0),
    'white': (255, 255, 255),
    'rand': tuple(random.randint(0, 255) for _ in range(3))
}
```

在 OpenCV 中，颜色是所有绘图函数的核心参数之一。使用字典形式集中管理颜色既方便调用，也使绘制结果更具一致性。如果需要更精准或风格化的颜色，可直接在配色网站选取并转换为 RGB/BGR 元组。

我给大家推荐一个我常用的颜色网站：[Palettes](https://flatuicolors.com/)，它分了很多组的常用 UI 颜色，并且每个组里面都设定了 HEX、RGB、RGBA 不同的颜色格式，适用于绝大多数场合。不过复制粘贴过来后，记得手动换一下 B 和 R 通道的位置哦。

### 1.2 绘制函数

OpenCV 的文本绘制函数是 ==cv2.putText()==，所有文字标注（如检测类别、坐标说明等）都依赖它完成。

```py
cv2.putText(img, text, org, fontFace, fontScale, color, thickness=None, lineType=None, bottomLeftOrigin=None)
```

img 表示需要绘制文本的图像。text 和 org 分别表示要绘制的文本（字符串）、文本 “左上角/左下角”的坐标。org 这个文本起始点位置，是由可选参数 bottomLeftOrigin 决定的，它是一个布尔值，若为 True，则文本原点在左下角（默认 False，在左上角）。

fontFace 表示字体类型（见 1.3 小节）。fontScale 表示字体缩放因子（决定字号，默认 1.0）。color 表示字体颜色（BGR 顺序）。thickness 表示线条粗细（即笔画宽度）。lineType 表示线型（cv2.LINE_4、cv2.LINE_8、cv2.LINE_AA）。

![1.1 文本绘制函数 - 参数可视化对比](/cv/ImageProcessing/08_xxx/01_text1.png)

从可视化结果可以看到，“字号” 与 “粗细” 是一个相对的关系，相同的粗细，字号越小看起来就越粗。所以如果要整体视觉上不发生变化，字号与粗细要一同变化。

### 1.3 字体类型

OpenCV 并不加载系统字体（不像 Pillow 的 ImageFont.truetype），而是使用内置的英文字体族，这些字体是 Hershey 系列（仅支持 英文字母和数字，不支持中文）：

- **cv2.FONT_HERSHEY_SIMPLEX(0)**：普通无衬线字体，默认字体，最常用
- **cv2.FONT_HERSHEY_PLAIN(1)**：简单字体，笔画较细，适合小字号
- **cv2.FONT_HERSHEY_DUPLEX(2)**：双线字体，比 SIMPLEX 更粗
- **cv2.FONT_HERSHEY_COMPLEX(3)**：带衬线字体，字体更复杂，视觉厚重
- **cv2.FONT_HERSHEY_TRIPLEX(4)**：三线字体，更宽、更立体感
- **cv2.FONT_HERSHEY_COMPLEX_SMALL(5)**：带衬线的小号字体，在小图中更易辨识
- **cv2.FONT_HERSHEY_SCRIPT_SIMPLEX(6)**：手写体（单线），类似手写英文
- **cv2.FONT_HERSHEY_SCRIPT_COMPLEX(7)**：手写体（多线），线条更浓、更艺术化
- **cv2.FONT_ITALIC(8)**：斜体标志（可与上述组合），通过位运算叠加

左图就是 OpenCV 中所有的默认字体，蓝色代表使用了斜体。右图是一个中文文本乱码与纠正的可视化显示，此处使用了 Pillow 进行绘制。

### 1.4 文本辅助函数

为了更好的绘制文本，OpenCV 还提供了几个相关的辅助函数，比如 ==cv2.getTextSize()==，它用于计算文本在当前字体/缩放/粗细下的像素尺寸和基线。一般用于做背景框、居中对齐、检查是否越界。

```py
(text_w, text_h), baseline = cv2.getTextSize(text, fontFace, fontScale, thickness)
```

- **text_w, text_h**：文本的像素宽高（不含基线以下的“尾巴”）。
- **baseline**：基线以下需要预留的像素（如字母 “g, y, j” 的下延部分）。

第二个函数是 ==cv2.getFontScaleFromHeight()==，在已知“想要的像素高度” 时，反求合适的 fontScale。你希望字号刚好为 24px、32px 这类“绝对像素高度”时非常好用。

```py
scale = cv2.getFontScaleFromHeight(fontFace, pixelHeight, thickness)
```

在 OpenCV 里，我们常常需要把文字绘制到某个矩形区域里，比如标签、按钮、图像标注等。那么问题来了，怎样才能让文字既不太小，也不溢出框外，还能居中显示？

为了解决这个问题，我们就做一个 🧪 小实验：定义一个矩形框，然后依次尝试控制文字的大小和位置，让它“正好”贴合在框内。要做到这一点，我们需要知道两个核心信息：

1. 文字在当前字体、大小下究竟占多大空间；
2. 给定框的尺寸后，应该选择多大的缩放比例 fontScale。

诶，因为矩形框已经定义好了，所以直接使用 getFontScaleFromHeight 推算出一个初步的 fontScale，然后再用 getTextSize 去测量文字的实际像素尺寸，这样就可以绘制了。

![1.3 文本绘制辅助函数 - 文本居中显示](/cv/ImageProcessing/08_xxx/03_text3.png)

通过结果中的 “步骤 3”，我们发现这样做虽然能让文字高度合适，但是当文字较长时，它的宽度可能会超出矩形框。原因很简单，getFontScaleFromHeight 只考虑高度，不关心宽度。于是，我们给算法加了一条 “宽度约束”：

```py
if text_width > target_width:
    scale *= target_width / text_width
```

这一行简单却有效：如果文字太宽，就按照宽度比例再整体缩小一次。最终，文字无论中英文、长短，都能稳稳落在框里。当然了，它还不是最优解，因为如果文本太长，还需要考虑到换行问题。不过，在这里就不过多展开了。

```py
# ---------- ② 自动根据高度计算 fontScale ----------
def auto_font_scale(font, target_h, thickness=2):
    if hasattr(cv2, "getFontScaleFromHeight"):
        return cv2.getFontScaleFromHeight(font, target_h, thickness)
    s = 1.0
    for _ in range(10):
        (_, th), _ = cv2.getTextSize("Hg", font, s, thickness)
        s *= target_h / (th + 1)
        if abs(th - target_h) < 1:
            break
    return s

# ---------- ③ 绘制文字在矩形框中 ----------
def draw_text_in_box(img, text, box, font, color=(0, 0, 0), thickness=2):
    x0, y0, x1, y1 = box
    h, w = y1 - y0, x1 - x0
    target_h, target_w = h - 20, w - 20

    # Step1: 根据高度算比例
    scale = auto_font_scale(font, target_h, thickness)

    # Step2: 测量文字大小
    (tw, th), bl = cv2.getTextSize(text, font, scale, thickness)

    # Step3: 如果太宽再缩放
    if tw > target_w:
        scale *= target_w / tw
        (tw, th), bl = cv2.getTextSize(text, font, scale, thickness)

    # Step4: 计算居中坐标
    tx = x0 + (w - tw) // 2
    ty = y0 + (h + th) // 2

    # 绘制文字
    cv2.putText(img, text, (tx, ty), font, scale, color, thickness, cv2.LINE_AA)
    return img
```

## 2. 绘制基础图形

OpenCV 中的基本图形，包括直线、矩形和圆形等。它们是最常见，也是最容易绘制的形状。绘制图形的第一步，与绘制文字一样，创建一个容纳的画布（3 通道以对应 BGR 图像）：

```py
img_fonts = np.ones((600, 800, 3), dtype=np.uint8) * 255  # 文本的白色画布
image = np.zeros((500, 500, 3), dtype=np.uint8)  # 图像这里我们使用黑色画布
```

OpenCV 提供的大多数绘图函数，都有共同的参数，在绘制文本时我们也见过：

- **img**：要绘制 图形/文本 的画布图像。
- **color**：用于绘制 图形/文本 的颜色（BGR 三元组）。
- **thickness**：正值为 图形/文本 轮廓线的粗细；否则颜色将填充形状。
- **lineType**：图形/文本 轮廓线的类型，cv.LINE_4（四连接线）、cv.LINE_8（八连接线）、cv.LINE_AA（抗锯齿线）。抗锯齿线使用高斯滤波算法，绘图质量更好，但速度较慢。另两个使用 Bresenham 算法绘制。
- **shift**：表示与定义图形的某些点的坐标相关的小数位数。

### 2.1 直线

绘制直线的函数为 ==cv2.line()==，函数的用法如下：

```py
cv2.line(img, pt1, pt2, color, thickness=1, lineType=8, shift=0)
```

pt1 与 pt2 分别表示二维坐标的起点和终点，会在画布的对应坐标之间，绘制一条直线将其连接起来。用一个实际例子来感受一下：

```py
image1 = np.zeros((500, 500, 3), dtype=np.uint8)
cv2.line(image1, (0, 0), (500, 500), COLORS['red'], 3)
cv2.line(image1, (0, 500), (500, 0), COLORS['orange'], 10)
cv2.line(image1, (250, 0), (250, 500), COLORS['yellow'], 3)
cv2.line(image1, (0, 250), (500, 250), COLORS['green'], 10)
```

### 2.2 矩形

绘制矩形的函数为 ==cv2.rectangle()==，函数的用法如下：

```py
cv2.rectangle(img, pt1, pt2, color, thickness=1, lineType=8, shift=0)
```

这里的 pt1 与 pt2 指的是，矩形左上角的坐标点和矩形右下角的坐标点，通过这两点来确定矩形坐标的位置。常用于检测框或裁剪区域可视化。还是来个具体例子：


## 3. 绘制高级图形

### 3.1 裁剪线

OpenCV 的绘制直线函数 cv2.line() 并不会自动“截断”线段，也就是说，你给它两个点 (x1, y1) 和 (x2, y2)，哪怕这两个点在图像外面，它也会尝试去画，常导致错误、性能变差，或者画出来一条很奇怪的长线。

为了避免这种情况，OpenCV 提供了 ==cv2.clipLine()== 函数，用来计算：**给定一条直线 + 一个矩形 → 求出线段与矩形的交点，并返回矩形内的可见部分**。函数的使用方法如下：

```py
retval, pt_1, pt_2 = cv2.clipLine(rect, pt1, pt2)
```

rect 表示裁剪的矩形区域，常常就是图像边界，如 (0, 0, width, height)。pt1 与 pt2 就是原始线段的起点和终点。返回值 retval 是一个布尔变量，表示是否有部分线段落在矩形内。pt_1 与 pt_2 表示裁剪后线段的新起点与新终点。

为了更直观的显示，我们就不画超出画布的直线了，在画布中取一个矩形，分别绘制两条直线，一条穿过矩形，一条不穿过，对于 retval 为真的线段区域，用加粗以及改变颜色的方式来显示：

![3.1 高级图形绘制 - 裁剪线](/cv/ImageProcessing/08_xxx/08_image.png =560x)

它通常用于：（1）绘制无限延伸的几何线（如霍夫直线检测结果）。（2）避免绘制超出图像的大范围线段，保证性能、避免溢出。（3）ROI 裁剪时，画 ROI 边界内的线段，比如检测线、辅助线等。

ROI = Region of Interest（感兴趣区域），直观的理解就是，你只关心图像中的一个矩形区域，其他区域可以忽略。

### 3.3 椭圆

OpenCV 中绘制椭圆的函数为 ==cv2.ellipse()==，椭圆由三个核心元素决定：中心点（ellipse center）、两个轴的长度（major axis、minor axis）、旋转角度（angle）。具体用法如下：

```py
cv2.ellipse(
    img,
    center,            # 中心点 (x, y)
    axes,              # 半径：(长半轴, 短半轴)
    angle,             # 椭圆旋转角度，顺时针计算

    lineType=cv2.LINE_8
)
```

### 3.4 多边形

一个多边形，就是由多个点按顺序连接形成的封闭图形：p0 → p1 → p2 → p3 → ... → p(n-1) → p0。OpenCV 的多边形点格式必须是：**一个 N×2 的 np.array**，且 dtype 必须是 **int32 (np.int32)**。绘制多边形的函数为==cv2.polylines()==，使用方法如下：

```py
cv2.polylines(img, pts, isClosed, color, thickness=1, lineType=cv2.LINE_8)
```

isClosed 参数表示是否自动闭合多边形，自动闭合会首尾相连。pts 表示点列表，外面要再套一层 list：

```py
pts = np.array([[x1, y1], [x2, y2], ...], np.int32)
pts = pts.reshape((-1, 1, 2))
############## pts 的两种方式 ##############
cv2.polylines(img, [pts], True, color, 2)
```

该函数在 “边缘与轮廓检测” 中已经使用过。==cv2.fillPoly()== 用于填充绘制好的多边形，==cv2.fillConvexPoly()== 是专门优化过的，用于填充凸多边形的函数。

```py
cv2.fillPoly(img, [pts], color)
cv2.fillConvexPoly(img, pts, color)
```

**凸多边形（Convex Polygon）** 是指任意两点连线，都完全落在多边形内部。**凹多边形（Concave Polygon）** 是指存在某两点连线，会跑到多边形外部。比如，五角星 ⭐️ 就是凹多边形。对于两种多边形的判定，还有一种方式，**扫描多边形的所有内角**：

- 所有内角 < 180° → 凸
- 有任何一个内角 > 180° → 凹

以三角形和五边形作为例子，分别绘制闭合/不闭合多边形，以及两种情况下的填充。fillPoly() 填充函数会自动闭合多边形，因此看不出什么区别。

### 3.5 shift 参数

OpenCV 很老的 API 是 C 写的，而 C 不喜欢在函数里传浮点数，所以设计了一个 trick：用整数模拟浮点，**真正坐标 = 输入整数坐标 / (2^shift)**，比如：

```py
pt = (100, 50)
shift = 1
真实坐标 = (100/2, 50/2) = (50.0, 25.0)
```

所以总结下来就是：shift 表示你给的坐标点被缩小了 2^shift 倍，用于支持亚像素（小数）精度。

- 如果 shift = 0 → 坐标按正常像素理解（整数）
- 如果 shift = 1 → 你的坐标会除以 2
- 如果 shift = 2 → 你的坐标会除以 4
- 以此类推…

因为绝大多数绘图坐标是整数像素，我们无需亚像素精度。所以，shift=0 是默认、也是最常用的设置。但也不是不会被用到，shift 主要用于“亚像素精度”。OpenCV 的设计是：你在用户端传整数，shift 表示小数位数，库内部帮你自动缩放回小数坐标。

```py
# 表示真实坐标 (10.5, 20.5)
cv2.circle(img, (21, 41), 3, (0,255,0), 1, shift=1)
```

因为 shift=1 → 除以 21 / 2 = 10.5，41 / 2 = 20.5。OpenCV 会把图形画在（10.5, 20.5）这个位置。
